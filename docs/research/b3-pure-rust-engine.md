# B3 research note — the pure-Rust sovereign engine + the llama.cpp debt-CLOSURE test

Status: **research note (informs #95/#90).** Research-first; no code proposed. This is the B3 deliverable the sovereignty plan (`docs/plans/sovereignty-plan.md`, branch `sovereignty-plan` @ `c51b5ec` — see Path note in §8) REQUIRES to exist **before M32 (#90) lands**: (a) the candle / mistral.rs / burn eval on a hard **zero-C-chain-including-the-tokenizer** axis, and (b) **the llama.cpp debt-closure test, defined here, concretely** — what is compared, on what inputs, what passes — plus the grep-enforced lane flip `engine=VENDORED-C-LLAMACPP → engine=PURE-RUST`. Raw survey data and per-source citations live in the sibling [`b3-pure-rust-engine-literature.md`](b3-pure-rust-engine-literature.md); this note is the reconciled design. ## 1. The decision frame (the plan's amendments, encoded)

- **Scope: a std-Rust HOST daemon, not no_std, never in-kernel.** Plan principle 2 ("the kernel never hosts an engine") is untouched: float/GPU/nondeterminism stay out of the guest; the engine serves `model:local/llama` over the M30 verified transport. The zero-C claim is ledger-scoped: **CLAIMED** = no compiled/vendored C/C++ anywhere in the daemon's cargo build graph, *including the tokenizer* (the axis most evals skip — §3); **ACCEPTED-PERMANENT** = the host glibc/Linux/rustc residual M31 already names (`host=RESIDUAL-TCB`); **out of scope** = GPU paths (B2's driver-VM problem, SOVEREIGNTY-ROADMAP §6), training, no_std.
- **Two debts, two fates — do not conflate them.** SOVEREIGNTY-ROADMAP §6's GPU/CUDA tax is `ACCEPTED-PERMANENT` (CUDA cannot be reimplemented; B2 quarantines it). The **vendored-llama.cpp CPU daemon (M32) is `TEMPORARY` debt** per plan principle 1, with this note as its grep-enforced closure plan. B3 closes the second debt only; the first stays quarantined forever, and the post-closure ledger says so.
- **The kernel seam is already built, proven, and CHEAP — which is exactly the trap.** `crates/tb-hal/src/infer.rs` makes engine identity a registry entry, not a call-site dependency: the 2-method object-safe `Sync` trait `InferBackend {scheme(), infer()}` (`infer.rs:163-168`), longest-prefix `resolve()` over each route's own `scheme()` literal (`:127`), the immutable `static ROUTES: &[&'static dyn InferBackend]` (`:206`). Agents hold an `INVOKE_MODEL` capability handle, never an engine name; swapping the daemon's engine changes **zero kernel bytes** — no kernel diff will ever betray whether llama.cpp is still there. *Because the swap is structurally free, never flipping the debt token is equally free.* Closure must therefore be a forced, graded, grep-checked event (§5–§6), or the debt is permanent by inertia.
- **Hard requirements from the plan:** GGUF + K-quant loading (**Q4_K_M is the load-bearing custody format** — requantizing through a Python/C toolchain re-opens the debt), CPU-only viability on the reference host, maintained, license-compatible, and a *mechanically provable* C-free chain (a "C-free if you set the right flags" engine counts only with a CI guard pinning the flags forever).

## 3. The load-bearing C finding — `onig` is in EVERY default candle build

HF `tokenizers` defaults are `["progressbar", "onig", "esaxx_fast"]`: `onig` = `onig_sys` = **oniguruma, a C regex library**; `esaxx_fast` = **C++** suffix-array. The pure-Rust backend (`fancy-regex`, wasm-proven; `compile_error!` if neither backend is chosen, so the choice is total and machine-visible) exists behind `default-features = false`. Verified chain status:

- **candle-core** (upstream main AND the rev mistral.rs pins) declares for non-wasm targets: `tokenizers = { workspace = true, features = ["onig"] }` → **every default candle/mistral.rs Linux build compiles onig_sys C today.** Cargo cannot unrequest a dependency-requested feature, so closure needs a **pinned one-line fork/`[patch]` of candle-core** (or the upstream PR making the backend a pass-through feature — the right long-term move). This is the ENTIRE C residue; after the patch it is grep-enforceable forever (§5 gate 1).
- **mistral.rs's own** `tokenizers = { version = "0.21.4", default-features = false }` already resolves C-free (lockfile-verified), and it additionally ships `gguf_tokenizer.rs` — a **pure-Rust tokenizer reconstruction from GGUF metadata** (`tokenizer.ggml.model`: llama/replit → Unigram, gpt2 → BPE). **Preferred custody shape:** ONE GGUF artifact carries weights AND tokenizer — one custody boundary, one hash in the M33 attestation, no `tokenizer.json` sidecar.
- Fallback tokenizer options, all pure-Rust: HF `tokenizers` with `fancy-regex` (the onig↔fancy-regex behavior-parity residual is NAMED and caught by §5 gate 2); `shimmytok` (GGUF-native with an llama.cpp parity suite — exactly on-target but young, audit-sized adopt-or-vendor candidate); `kitoken`; `tiktoken-rs`/`tekken-rs` (family-specific). **Anti-pattern, do not copy:** HF `text-embeddings-inference` sets `default-features=false` then re-enables `["onig","esaxx_fast"]`.

## 4. Recommendation + runner-up

**Recommendation: `mistral.rs` (MIT), with the §3 one-line candle-core patch pinned and the §5 deny-grep enforcing it forever.** It is the only *maintained* engine that is simultaneously GGUF-k-quant-native, CPU-only-viable, **server-shaped for the M30/M32 daemon seam**, tokenizer-self-contained from the GGUF artifact, and same-day active. Named risks carried on the witness, not hidden: single-lead bus factor; 70+-crate audit surface (mitigated by `--locked` + the lockfile deny-grep); CPU decode ~1.5–3× slower than llama.cpp (the §5 perf floor makes the regression budget explicit).

**Runner-up: candle direct** (`candle-core` + `candle-transformers` quantized pipeline, Apache-2.0/MIT, HF-backed): strictly smaller audit surface, stronger institutional maintainer — better if the daemon serves one pinned model family and we write the serving loop ourselves. **Same patch, same closure test; swapping recommendation and runner-up changes no token and no gate**, so the call can be re-taken at implementation time without re-research. Track `oxillama` as the by-construction zero-C endpoint (§2 watch item; audit its bundled tokenizer before any adoption). **Not now:** burn (no GGUF — re-eval if #1187 lands), ratchet (web-first), rustformers/llm (archived AND C-bound), lm.rs (dead), llama-gguf (unproven).

## 5. THE DEBT-CLOSURE TEST (defined HERE, before M32 lands — the plan's load-bearing deliverable)

M32's witness is REQUIRED (plan §M32) to carry `engine=VENDORED-C-LLAMACPP debt=SOVEREIGNTY-OPEN-B3 memsafety=UNSAFE-C-PROCESS-CONFINED weights=UNTRUSTED-INPUT-NAMED` — the debt machine-checked, not prose-remembered. **Closure = all five gates green in ONE CI run**, then the §6 lane flip. The pinned constants (ppl margin, perf floor, corpus hashes, reference host) are frozen in the M32 proposal DoD before any engine code merges — the M24 lower-bound-then-refuse model: the gate REFUSES (stays `VENDORED-C-LLAMACPP`) on insufficient evidence rather than over-claiming parity.

1. **Zero-C chain proof — `cchain=CC-POISON-VERIFIED`.** Two independent mechanisms, both fail-closed: (a) **poisoned-toolchain build** — the daemon workspace builds `cargo build --locked --release` with `CC=/bin/false CXX=/bin/false`, so any build script invoking a C/C++ compiler fails the build; success IS the proof; (b) **lockfile deny-grep** in both run scripts — the daemon `Cargo.lock` must match none of `onig_sys|cmake|bindgen|esaxx|cc"` -class patterns (tuned to zero false positives at implementation, fail-closed). This is what makes the §3 candle-core patch permanent, not aspirational.
2. **Tokenizer parity — `tokparity=LLAMACPP-EXACT` (exact, because exactness is honest here).** Inputs: a **hash-pinned corpus, ≥1k lines** (multilingual + code + edge cases: empty/whitespace runs, emoji/ZWJ, CJK, byte-fallback, BOS/EOS), against the **same GGUF artifact actually served**. Compared: full token-id sequences from the pure-Rust tokenizer vs `llama.cpp tokenize`. Pass: **byte-exact equality on 100% of lines.** Tokenization is discrete — exact equivalence is gateable, and this gate is precisely what catches the §3 onig→fancy-regex behavior residual.
3. **Model quality bar — `ppl-delta=<measured>` + `quality=FLOOR-MET` (statistical, NOT output-equivalence).** The plan offered *"output-equivalence or a quality bar"*; **output-equivalence on decode is considered and REJECTED as dishonest** — greedy decode legitimately diverges across engines (kernel order, float rounding); an argmax-exact gate either never passes or passes vacuously. (This supersedes the `equiv=ARGMAX-EXACT` gate in an earlier draft of this note.) The honest bar: **perplexity over a hash-pinned frozen eval slice, same GGUF file in both engines, pure-Rust within ≤0.5% relative of llama.cpp's.** Greedy prefix-agreement length over a pinned prompt set is RECORDED on the witness, never gated. The harness (not a human) computes the verdict and emits `quality=FLOOR-MET|FLOOR-NOT-MET`.
4. **Perf floor — `tps-ratio=<measured>` + `perf=FLOOR-MET` (anti-regression, deliberately loose).** Same GGUF, same pinned reference host, same pinned thread count (pinning float reduction order), pinned prompt/decode length: pure-Rust CPU decode **≥ 1/3 of llama.cpp t/s**. Encodes the measured 1.5–3× class so closure cannot smuggle in a lane-timeout regression. No parity claimed — floor only.
5. **The grep-enforced lane flip (verbatim from the plan's closure DoD).** The M32 lane witness flips `engine=VENDORED-C-LLAMACPP` → `engine=PURE-RUST`, the `debt=SOVEREIGNTY-OPEN-B3` token is RETIRED, `memsafety=UNSAFE-C-PROCESS-CONFINED` upgrades to `memsafety=SAFE-RUST-DEPS-UNAUDITED` (deps carry unsafe Rust — forbid-unsafe is NOT claimed; "no C parser on untrusted weights" IS), **AND both run scripts gain a REJECT for `VENDORED-C` on that lane** — after which a llama.cpp reappearance is a CI failure, not a memory. Mechanics in §6.

**Post-closure witness line** (one line; every field positively required by the run scripts in this order, so the token cannot be flipped by editing a string — the corroborating evidence must be on the SAME line):

```
M32: local-infer OK engine=PURE-RUST cchain=CC-POISON-VERIFIED tokparity=LLAMACPP-EXACT ppl-delta=<measured>% quality=FLOOR-MET tps-ratio=<measured> perf=FLOOR-MET gguf=SHA256:<64hex> weights=UNTRUSTED-INPUT-NAMED memsafety=SAFE-RUST-DEPS-UNAUDITED
```

**Token reconciliation [BEYOND]:** the artifact pin is its own field (`gguf=SHA256:<64hex>`) and does NOT replace the plan-required `weights=UNTRUSTED-INPUT-NAMED` — identity-pinning a weights file does not make its contents trusted; an earlier draft's `weights=PINNED-SHA256:` would have silently clobbered a required honesty token. **Scope honesty:** closure verifies the ENGINE behind the already-proven M30 transport leaf (transport not re-verified); bit-for-bit identity across all inputs is NEVER claimed; model semantics are out of scope; the closure gates are **host-side measurements, honestly NOT Kani proofs** — any new tb-encode codec leaf they introduce (e.g. a closure-witness/ledger frame consumed by the M33 attestation) bumps `EXPECTED_HARNESSES` (`scripts/verify-encode.sh:203`, =80 at writing; M29 khash takes it to 84) in LOCKSTEP per the standing tamper-evidence rule.

## 6. The grep-enforced token flip — four coordinated guards

One lane-token value change on one witness line, exactly the M29 `mac=` / M30 `transport=` convention. Each guard reuses an EXACT in-tree pattern:

1. **REJECT-for-VENDORED-C** (the plan's closure DoD, the headline guard). Once closure lands, both run scripts do `grep -qF -- 'engine=VENDORED-C-LLAMACPP'` on the M32 lane log and **FAIL if found** (also rejecting a stray `debt=SOVEREIGNTY-OPEN-B3`) — the mirror of the M28 skip-reject at `run-x86_64.sh:342`. Reverting closure trips it. **BEFORE closure the inverse holds:** `VENDORED-C-LLAMACPP` is the EXPECTED honest state, and `engine=PURE-RUST` *without* the §5 corroborating fields is rejected.
2. **TOKEN-REQUIRE** (positive anti-hollow). `grep -qE -- 'M32: local-infer OK engine=PURE-RUST cchain=CC-POISON-VERIFIED tokparity=LLAMACPP-EXACT ppl-delta=[0-9.]+% quality=FLOOR-MET tps-ratio=[0-9.]+ perf=FLOOR-MET gguf=SHA256:[0-9a-f]{64}'` MUST match else FAIL "hollow closure pass" — the M28 positive-require pattern at `run-x86_64.sh:346`.
3. **OVERCLAIM-REJECT with strip-before-grep.** On lines near the M32/infer marker, FIRST strip the benign honesty tokens (`PURE-RUST`, `VENDORED-C-LLAMACPP`, `SHA256:[0-9a-f]*`, `SAFE-RUST-DEPS-UNAUDITED`), THEN case-insensitively reject `sovereign|native-cuda|zero-linux|full-parity|bit-identical` — so the guard never false-positives on the very token that DECLARES the state. Identical to M28's `sed -e 's/KEYED-NONCRYPTO//g' …` strip-then-grep at `run-x86_64.sh:358-360`.
4. **Forgery-proofing (the plan's M31 injection amendment, inherited).** The closure witness is emitted by the **harness/run-script side**, never echoed through model output; untrusted model bytes on the M32 lane stay hex-encoded/regex-inert before serial, because the grep guards are not line-anchored and model text could otherwise forge the very marker that closes the debt.

**Ledger downgrade in lockstep:** the sovereignty-ledger row for vendored llama.cpp goes `TEMPORARY` (M32 lands, debt open) → **`CLOSURE-DEFINED`** (this note + the M32 proposal DoD pin the gates and constants) → **`CLOSED`** (both run scripts' M32 lanes print the §5 witness green). The §6-roadmap GPU/CUDA row stays `ACCEPTED-PERMANENT`/`QUARANTINED` — B3 never claims to close it. **Net: "is llama.cpp gone?" is answerable by `grep engine=` on the green logs, and un-closing the debt is a hard red.**

## 7. CI determinism — dual-lane

- **Lane A (default cumulative boot, every PR):** the deterministic in-kernel `MockBackend` (`prompt ^ 0xA110_C0DE`, `infer.rs:191` — no clock/rng/IO; QEMU has no GPU/net) stays; the M0..M28 cumulative-marker boot remains bit-reproducible and carries NO `engine=` claim (M31's chain variant stays `backend=MOCK-DETERMINISTIC`). The engine swap can never destabilize it.
- **Lane B (the M32/closure lane — the ONLY place real weights appear):** determinism-by-pinning. Weights are a SHA-256-addressed cached CI artifact (**never an unpinned download** — exactly the nondeterminism + supply-chain debt the ledger forbids), `temp=0` greedy, fixed seed, fixed thread count; golden comparisons are then bit-stable run-to-run on the same host. The engine (vendored-C or pure-Rust) runs as a **host-side peer behind the M30 channel** (`transport=TB-VMM-HOST`; the aarch64 lane's host-peer gap is the M30-named external-dependency decision), so in-guest boot determinism is untouched either way. The lane's witness line is the SOLE authority for flipping `engine=`.
- **Honest-skip is a FAILURE.** If pinned weights are unavailable, the lane prints `engine=VENDORED-C-LLAMACPP cchain=NOT-RUN` and FAILS closure — never a silent pass (the both-arches no-hollow-skip rule).
