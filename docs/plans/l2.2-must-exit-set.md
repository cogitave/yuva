# L2.2 — full MUST-handle VM-exit set: the tb-core exit demultiplexer (CPUID/MSR/CR/HLT/EPT/ext-int/triple-fault) + VMCALL hypercall seam

> **Task #43** · DoD marker: `L2.2: exits OK` · Effort: **L** · Slug: `l2.2-must-exit-set`
> **Depends on:** #7, #6, #2

> Auto-generated from the `tabos-forward-plans` ultracode planning workflow (research-backed, decision-closed). Treat as the working spec for this task; refine in place as implementation reveals detail.

## Goal
L2.0 catches exactly one CPUID exit via a single-shot VMLAUNCH that never resumes; L2.1 adds the EPT-violation demand arm. L2.2 builds the real multi-exit demultiplexer: a VMLAUNCH→VMRESUME run loop with full guest-GPR save/restore, plus the minimum-correct handler for every Intel-SDM MUST exit reason — CPUID (forge), RDMSR/WRMSR (MSR-bitmap-gated forge), control-register access for CR3/CR0/CR4 (emulate), HLT, EPT violation (map+resume), external-interrupt (reflect to host), triple-fault (fatal teardown), and VMCALL wired as the tb-core hypercall entry — each tagged reflect/emulate/inject and each obeying the RIP-advance discipline. Done = a deliberately-crafted nested guest triggers every MUST exit, each is dispatched correctly, the observed-reason coverage mask is asserted to cover the MUST set, and the cumulative serial marker "L2.2: exits OK" prints on BOTH the real nested-VMX lane (genuine walk) and the graceful-skip lane (no VMX), with all new silicon-unsafe confined to crates/tb-hal/src/arch/x86_64/vmx/ so the kernel stays #![forbid(unsafe_code)].

## Approach
1. 1. GprState + run loop foundation (world_switch.rs). Add `#[repr(C)] struct GprState { rax,rbx,rcx,rdx,rsi,rdi,rbp,r8..r15: u64 }` (RSP/RIP/RFLAGS live in the VMCS, not here). Const-assert its field offsets; the new asm stub uses those literal offsets. Replace the single-shot launch with two entry points sharing one exit stub: `__tabos_vmx_run(gprs: *mut GprState, resume: u8) -> u64`. Entry: push host callee-saved (rbp,rbx,r12-r15), push the gprs pointer, VMWRITE HOST_RSP=rsp + HOST_RIP=exit-label, load guest GPRs from *gprs (rax last via a scratch slot), then `vmlaunch` if resume==0 else `vmresume`. Exit-label (HOST_RIP): immediately `push rax` then recover the gprs pointer from the known host-stack slot, store rbx..r15 + the pushed rax into *gprs, pop host callee-saved, return RFLAGS-derived status (0=clean VM-exit, 1=VMfail). This is the Barbervisor/KVM pattern; the CPU does NOT preserve RAX..R15 across the switch (SDM 27.5.1), so the stub owns it.
2. 2. Vcpu state struct (mod.rs). Introduce `struct Vcpu { gprs: GprState, launched: bool, ept: EptMap, gp: GuestPaging, seen_mask: u32, exit_count: u32 }`. `seen_mask` is the coverage accumulator (one bit per tracked MUST reason); `exit_count` is the runaway-loop watchdog (cap 64 → fatal). The Rust-side run loop: `loop { ok = world_switch::run(&mut gprs, launched); launched=true; if !ok { return EntryFailed{vm_error: exit::vm_instruction_error()} } match dispatch::handle(&mut vcpu) { Continue=>{}, Done=>break, Fatal(r)=>break } }`.
3. 3. MSR bitmap (msr_bitmap.rs). Allocate one 4 KiB frame, zero it (= in-range MSRs pass through hardware). Provide `fn set_read_exit(bm, msr)` / `set_write_exit(bm, msr)` that place the bit per SDM 25.6.9 layout: read-low 0x000-0x3FF (MSR 0..0x1FFF), read-high 0x400-0x7FF (0xC0000000..0xC0001FFF), write-low 0x800-0xBFF, write-high 0xC00-0xFFF. Set the read+write bits for ONE sentinel MSR (decided: 0x4B564D00, a KVM-style paravirt-range index that is otherwise unimplemented) so exactly that RDMSR/WRMSR traps while real MSRs the kernel may read pass straight through. VMWRITE MSR_BITMAP (0x2004) = frame PA.
4. 4. Controls (controls.rs). Extend `Controls`/`adjusted()` desired bits: primary procbased gains HLT-exiting(bit7) | CR3-load-exiting(bit15) | CR3-store-exiting(bit16) | use-MSR-bitmaps(bit28) | activate-secondary(bit31); pin-based gains external-interrupt-exiting(bit0); VM-exit controls gain acknowledge-interrupt-on-exit(0x8000,bit15) on top of host-addr-space-size|save/load-EFER; secondary keeps enable-EPT. CRITICAL: after the (desired|allowed0)&allowed1 clamp, verify each REQUIRED desired bit survived; if the substrate masked one off, record it so the corresponding arm degrades to a logged honest gap rather than a false pass.
5. 5. CR guest/host masks (mod.rs program_vmcs). To force CR0/CR4 write exits deterministically: set CR0_GUEST_HOST_MASK to own a benign bit (decided CR0.MP bit1) and CR4_GUEST_HOST_MASK to own CR4.DE(bit3); seed CR0_READ_SHADOW/CR4_READ_SHADOW = the guest's actual CR0/CR4. A guest `mov crN, rax` that flips an owned bit then mismatches the shadow → reason-28 exit. CR3 needs no mask (the load/store-exiting controls cover it).
6. 6. Exit readers (exit.rs). Promote the dead-code readers to live and add: `vm_exit_intr_info()` (VM_EXIT_INTR_INFO 0x4404; vector=bits7:0, type=bits10:8, valid=bit31), `decode_cr_qual(q)` → (cr_num=bits2:0, access_type=bits5:4, gpr=bits11:8) per Table 28-3, and `guest_rip()`/`set_guest_rip()` over GUEST_RIP(0x681e). Keep exit_reason/qualification/instruction_len/guest_physical_address.
7. 7. RIP-advance discipline as a type (dispatch.rs). Each arm returns `enum Step { Advance, NoAdvance, Done, Fatal }`; the loop advances GUEST_RIP by VM_EXIT_INSTRUCTION_LEN(0x440c) iff `Advance`. Advance for the trapping instructions (CPUID, RDMSR, WRMSR, MOV-CR, VMCALL, I/O); NEVER advance for EPT-violation/exception (`NoAdvance`, re-execute after the fix) or external-interrupt; `Done`/`Fatal` tear down. This encodes the design's #1 correctness pitfall structurally instead of ad-hoc.
8. 8. CPUID forge (dispatch.rs, reason 10 = EMULATE). Execute the real host CPUID for (RAX leaf, RCX subleaf) from gprs, then sanitize: on leaf 1 clear ECX.VMX(bit5) (deny the nested guest its own VMX) and set ECX.hypervisor-present(bit31); on leaf 0x40000000 return a 'TBOSTBOSTBOS' vendor signature + max-leaf in EAX. Write EAX/EBX/ECX/EDX back into gprs; Step::Advance. CPUID is an UNCONDITIONAL exit (SDM 25.1.2) so it needs no control bit.
9. 9. RDMSR/WRMSR forge (dispatch.rs, reasons 31/32 = EMULATE). On RDMSR of the sentinel: return a known magic in EDX:EAX (gprs.rdx:gprs.rax). On WRMSR of the sentinel: validate+swallow (log value EDX:EAX from gprs). Any other (out-of-range) MSR that exits is reflected by re-reading/writing the real MSR. Step::Advance.
10. 10. CR-access (dispatch.rs, reason 28 = EMULATE). Decode the qualification: for MOV-to-CR3/CR0/CR4 (type 0) take the source value from gprs[gpr] and VMWRITE GUEST_CRn + (for CR0/CR4) the matching READ_SHADOW; for MOV-from-CR3 (type 1) write GUEST_CR3 into gprs[gpr]. (No store-exiting exists for CR0/CR4, so only CR3 reads can trap — documented.) Step::Advance.
11. 11. HLT (reason 12). In the walk guest HLT is the terminal success sentinel → log + Step::Done (clean break, frames freed by the caller). Document the general-guest semantics (set GUEST_ACTIVITY_STATE=HLT/1, resume on next interrupt) for L2.4; the L2.2 proof uses terminal-halt.
12. 12. External-interrupt (reason 1 = REFLECT-TO-HOST). Mechanism: just before one VMRESUME, fire a self-IPI via the M8 LAPIC ICR (fixed delivery, self shorthand, vector 0x40). With pin-based ext-int-exiting set the pending interrupt forces a reason-1 exit irrespective of guest RFLAGS.IF; with ack-on-exit the CPU latches the vector into VM_EXIT_INTR_INFO. Handler: read the vector, EOI the LAPIC (host keeps ownership — reflect, do NOT inject into the guest), mark coverage, Step::NoAdvance (asynchronous). Watchdog guards against non-delivery.
13. 13. Triple-fault (reason 2 = FATAL) in a SEPARATE micro-guest. Because a triple fault makes the VMCS unusable, run a second tiny world-switch session whose guest has GUEST_IDTR_LIMIT=0 and executes INT3 → #BP→#DF→triple fault → reason-2 exit. Handler: log + Step::Fatal teardown (VMXOFF, frame-free). This exercises the fatal arm without corrupting the main walk's VMCS.
14. 14. VMCALL hypercall (hypercall.rs, reason 18 = EMULATE). VMCALL is unconditional. ABI (decided, KVM-compatible for the eventual L2.3 device-model port): RAX=hypercall number, args in RBX/RCX/RDX/RSI, return in RAX. Numbers: 0 TBHC_NULL (returns magic 0x54424F53 'TBOS' — the proof's ping), 1 TBHC_CONSOLE_PUTC (RBX=byte → host serial, foreshadows the L2.3 seam), 2 TBHC_EXIT (RBX=code → Step::Done); unknown → RAX=-1 (ENOSYS), Step::Advance. The guest's ping VMCALL returns the magic; a wrong RAX restore makes the magic-check fail loudly.
15. 15. Facade + coverage proof (lib.rs). Add `pub struct VmxExitProof { pub seen_mask: u32, pub ok: bool, pub gaps: u32 }` and `pub fn vmx_exit_walk() -> VmxExitProof` (x86_64 drives the two sessions; aarch64 returns ok=true,seen=0 as NotApplicable). MUST mask = CPUID|MSR_R|MSR_W|CR|HLT|VMCALL|EPT|EXTINT|TRIPLE; ok = (seen_mask & MUST)==MUST (or, on a control-clamped substrate, MUST minus the honestly-logged gaps).
16. 16. Kernel render (kernel/src/main.rs). After the L2.0 block, call vmx_exit_walk(); print one `vmx-exit: reason=NN rip=0x.. <reflect|emulate|inject>` line per handled exit plus a `vmx: exits seen = CPUID MSR-R MSR-W CR HLT VMCALL EPT EXTINT TRIPLE` summary; then `L2.2: exits OK` on ok, `L2.2: exits OK (vmx unavailable, skipped)` on the no-VMX/skip path, `L2.2: exits OK (x86-only, n/a on aarch64)` on aarch64, and a non-OK diagnostic line (no marker substring) on a genuine coverage failure.
17. 17. Tests + CI. Add host unit tests (#[cfg(test)]) for the pure encoders (CR-qual decode, MSR-bitmap bit placement, CPUID-forge sanitizer, intr-info parse, hypercall decode). Extend .github/workflows/l2-nested-vmx.yml to additionally grep `^L2\.2: exits OK$` and the coverage summary on the KVM -cpu host lane; ci.yml (TCG) asserts the skip-substring. Update SOVEREIGNTY-L2-ROADMAP §7/§8 marking L2.2 implemented.

## Decisions — no open questions (açık karar bırakma)
- Dispatch table classification (reflect/emulate/inject), final: CPUID(10)=emulate-forge; RDMSR(31)/WRMSR(32)=emulate-forge gated by MSR bitmap; CR-access(28)=emulate; HLT(12)=emulate (terminal-halt in the proof); EPT-violation(48)=emulate/map+resume (the L2.1 arm, subsumed here); external-interrupt(1)=reflect-to-host (EOI on host, not injected); triple-fault(2)=fatal teardown; VMCALL(18)=emulate as hypercall; EPT-misconfig(49)=fatal; any unknown reason=fatal-with-diagnostic. The 'inject' path (VM-entry interruption-information) is implemented but used only by the EPT/exception future arms, not the trivial guest.
- Emulated vs reflected split: everything the hypervisor must SEE BUT THE GUEST MUST NOT (CPUID leaves, sentinel MSRs, owned CR bits, VMCALL, EPT faults) is EMULATED in tb-core; only asynchronous host-targeted external interrupts are REFLECTED to the host IDT/LAPIC; fatal conditions (triple-fault, EPT-misconfig, unknown) tear down. No MUST exit is silently passed through.
- RIP-advance is a returned enum (Advance/NoAdvance/Done/Fatal) per arm, not an ad-hoc bool — advance only after trapping instructions, never after EPT-violations/exceptions/external-interrupts. A 64-exit watchdog converts any accidental no-advance infinite loop into a loud failure.
- Hypercall ABI: KVM-compatible register layout (RAX=nr, RBX/RCX/RDX/RSI=args, RAX=ret). Rationale: the L2.3 device-model demotion reuses tb-vmm's KVM-shaped exit loop, so matching KVM's hypercall convention minimises the eventual port. Initial numbers 0=NULL/ping, 1=CONSOLE_PUTC, 2=EXIT; unknown→ENOSYS(-1). Reserves room for the §2 vm_create/vcpu_run/mem_grant/irq_inject/iommu_map set.
- MSR interception policy: enable use-MSR-bitmaps with an all-zero bitmap (in-range MSRs pass through hardware) and trap exactly ONE sentinel MSR (0x4B564D00) read+write — this exercises the bitmap path precisely and deterministically without trapping every MSR the M0-M18 kernel reads, while proving selective interception works.
- CR-exit elicitation: trap CR3 via CR3-load+store-exiting controls; trap CR0/CR4 writes via owning a benign bit each (CR0.MP bit1, CR4.DE bit3) in the guest/host masks with seeded read shadows. Accepted limitation (documented): MOV-from-CR0/CR4 cannot be trapped (no store-exiting for CR0/CR4), so only CR3 reads exit on the read side.
- External-interrupt is elicited deterministically by a self-IPI (LAPIC ICR, fixed, self-shorthand, vector 0x40) fired immediately before a VMRESUME, with pin-based external-interrupt-exiting + VM-exit acknowledge-interrupt-on-exit so the vector lands in VM_EXIT_INTR_INFO regardless of guest IF. The host stays cli'd in root mode so the IPI is taken only as a non-root VM-exit. Fallback (documented): a short LAPIC one-shot timer if a substrate delays IPI self-delivery.
- Triple-fault runs as its OWN second world-switch session (separate VMCS, IDTR limit 0, INT3) because a triple fault makes the originating VMCS unusable; it must not be the last step of the main trapping-walk guest.
- Code lands in tb-hal/arch/x86_64/vmx + the safe lib.rs facade (NOT yet extracted to crates/tb-core). tb-core extraction is explicitly L2.3 when the device-model seam forms; pulling it forward would duplicate churn. The kernel stays #![forbid(unsafe_code)] and only branches on VmxExitProof.
- Proof artefact: a u32 coverage bitmask asserted to cover the MUST set, rendered as a per-exit serial trace + a named summary line. Dual-marker discipline (real walk vs graceful-skip vs aarch64-n/a) mirrors L2.0 exactly so hosted TCG CI stays green while the genuine proof is gated to the nested-VMX lane.
- Guest-GPR transport: a #[repr(C)] GprState with const-asserted offsets shared between the asm stub and Rust; RAX is saved/restored via a scratch slot since it is the stub's working register. A software round-trip unit test plus the hypercall-ping magic guard both catch a mis-wired save/restore.
- On a substrate whose control-MSR clamp removes a REQUIRED control bit (so an exit cannot be elicited), the affected arm is recorded as an honest logged gap and subtracted from the asserted MUST set for that run — never silently counted as covered. The bare marker prints only when the achievable MUST set is fully covered.

## Risks & mitigations
- Guest-GPR save/restore in the new asm stub is the highest-risk surface: one wrong offset corrupts a forged CPUID/MSR result or diverges the guest. Mitigation: #[repr(C)] GprState with const-asserted offsets used as literals in the asm, a software round-trip unit test, and the hypercall-ping magic (0x54424F53) which fails loudly if RAX restore is wrong.
- RIP-advance errors: advancing after EPT-violation/external-interrupt re-executes/skips wrongly; forgetting to advance after a trapping instruction infinite-loops. Mitigation: advance is a per-arm returned enum (not ad-hoc), and a 64-exit watchdog turns any runaway into a loud Fatal instead of a hang.
- VMLAUNCH-vs-VMRESUME confusion across the loop (first entry must VMLAUNCH, subsequent must VMRESUME) → VMfail with error 5 (VMRESUME-nonlaunched) or 4 (VMLAUNCH-nonclear). Mitigation: a `launched` flag in Vcpu and reading VM_INSTRUCTION_ERROR(0x4400) on every VMfail to localise (surfaced as EntryFailed{vm_error}).
- External-interrupt non-determinism: a self-IPI may not be delivered in time under a given nested substrate, leaving reason-1 uncovered. Mitigation: pin-based ext-int-exiting forces ANY pending host interrupt to exit; arm both a self-IPI and a short LAPIC one-shot; if neither fires within the watchdog, record an honest logged gap (documented) rather than a silent false pass — the bare marker is withheld unless the achievable MUST set is fully covered.
- Triple-fault corrupts its VMCS, so sequencing it inside the main walk would break later resumes. Mitigation: dedicated second session with its own VMCS and a teardown-only handler.
- Control-MSR adjust may strip a required control (e.g. a substrate that disallows CR3-store-exiting or ack-on-exit), making an arm un-eliciteable. Mitigation: post-clamp verification of each required bit; un-survivable controls degrade their arm to a logged gap, not a fake pass.
- Nested-VMX is not guaranteed on hosted GitHub runners (the persistent L2.0 risk), so the genuine walk can only run on the self-hosted/bare-metal lane. Mitigation: same KVM_OK allow-skip gating; the skip path still prints the marker substring to keep ci.yml green, and the real proof is the l2-nested-vmx job / Arda's hardware.
- MSR-bitmap mis-layout (swapping read/write or low/high sub-regions) would trap the wrong MSRs and potentially fault the host kernel's own MSR reads. Mitigation: the bitmap setter is unit-tested against the SDM 25.6.9 byte layout and only the single sentinel MSR is armed; all real in-range MSRs pass through untrapped.

## Research notes (with sources)
- Basic VM-exit reason numbers (Intel SDM Vol 3D Appendix C 'VMX Basic Exit Reasons', cross-checked against Linux arch/x86/include/uapi/asm/vmx.h): EXTERNAL_INTERRUPT=1, TRIPLE_FAULT=2, CPUID=10, HLT=12, VMCALL=18, CR_ACCESS=28, IO_INSTRUCTION=30, MSR_READ(RDMSR)=31, MSR_WRITE(WRMSR)=32, EPT_VIOLATION=48, EPT_MISCONFIG=49, PREEMPTION_TIMER=52. The basic reason is bits 15:0 of VM_EXIT_REASON(0x4402).
- CPUID and VMCALL cause UNCONDITIONAL VM exits in non-root operation (SDM Vol 3C §25.1.2/§25.1.3 instruction-behaviour tables); HLT, RDMSR, WRMSR, MOV-to/from-CR, and I/O are CONDITIONAL, gated by their VM-execution controls — so eliciting them requires explicitly enabling the matching control bit.
- MSR-bitmap structure (SDM Vol 3C §25.6.9 'MSR-Bitmap Address'): a 4 KiB page of four 1 KiB sub-bitmaps — read-low bytes 0x000-0x3FF (MSRs 0x00000000-0x00001FFF), read-high 0x400-0x7FF (0xC0000000-0xC0001FFF), write-low 0x800-0xBFF, write-high 0xC00-0xFFF. With 'use MSR bitmaps' (primary procbased bit 28) set, an in-range MSR causes a VM-exit only if its bit is set; out-of-range MSRs always exit. Field MSR_BITMAP=0x2004 (confirmed Linux asm/vmx.h).
- CR-access exit qualification (SDM Vol 3C Table 28-3, confirmed Linux asm/vmx.h CONTROL_REG_ACCESS_*): bits 2:0 = control-register number, bits 5:4 = access type (0=MOV to CR, 1=MOV from CR, 2=CLTS, 3=LMSW), bits 11:8 = source/dest GPR (0=RAX..15=R15), bits 31:16 = LMSW source data. EXIT_QUALIFICATION=0x6400.
- Control bit positions (Linux asm/vmx.h, SDM Tables 25-6/25-7): primary procbased HLT-exiting=bit7, CR3-load-exiting=bit15, CR3-store-exiting=bit16, use-MSR-bitmaps=bit28, activate-secondary=bit31; pin-based external-interrupt-exiting=bit0; VM-exit control acknowledge-interrupt-on-exit=0x00008000 (bit15), save/load-EFER=0x00100000/0x00200000, host-addr-space-size=0x00000200. CR3 needs load/store-exiting to trap; CR0/CR4 writes trap only when modifying host-owned bits per CR0/CR4_GUEST_HOST_MASK(0x6000/0x6002) vs CRn_READ_SHADOW(0x6004/0x6006) (SDM §25.6.6).
- External-interrupt path: when pin-based 'external-interrupt exiting' is 1, external interrupts cause VM exits and are not blocked by guest RFLAGS.IF; with VM-exit control 'acknowledge interrupt on exit' the CPU acknowledges the interrupt controller and stores the vector in VM_EXIT_INTR_INFO(0x4404) with valid bit31, type bits10:8, vector bits7:0 (SDM Vol 3C §25.2 + Table 28-... VM-exit interruption-information; Linux INTR_INFO_* masks).
- Event injection (the 'inject' option): VM-entry interruption-information field(0x4016) — vector 7:0, interruption type 10:8 (0=external interrupt, 2=NMI, 3=hardware exception, 6=software exception), deliver-error-code bit11, valid bit31; paired with VM_ENTRY_EXCEPTION_ERROR_CODE(0x4018) and VM_ENTRY_INSTRUCTION_LEN(0x401a) (SDM Vol 3C §27.6).
- RIP-advance source of truth: VM_EXIT_INSTRUCTION_LEN(0x440c) gives the byte length of the instruction that caused a fault-like exit; software-trapped instructions (CPUID/MSR/MOV-CR/HLT/VMCALL/IO) leave GUEST_RIP(0x681e) AT the instruction → advance by that length, while EPT-violations and exceptions leave RIP positioned to re-execute → never advance (SDM Vol 3C §28.2.1/§28.2.2).
- HLT/activity state: GUEST_ACTIVITY_STATE(0x4826) value 1 = HLT state; VMX_MISC bit6 advertises HLT activity-state support (Linux VMX_MISC_ACTIVITY_HLT=BIT(6)); a guest may be (re)entered in HLT state to model halt-until-interrupt — used for the general L2.4 guest, while L2.2's proof uses terminal-halt-as-success.
- VMfail diagnostics: VM_INSTRUCTION_ERROR(0x4400) enumerates causes — 4=VMLAUNCH non-clear VMCS, 5=VMRESUME non-launched VMCS, 7=invalid control field, 8=invalid host-state field (Linux vm_instruction_error_number; SDM Vol 3C §31.4) — read on every VMfail to localise a control/state bug instead of failing silently.
- Hypercall ABI precedent: KVM uses RAX=hypercall number, RBX/RCX/RDX/RSI=args, return in RAX (Documentation/virt/kvm/hypercalls.rst). NOVA/Hedron implement a capability-based portal VMCALL hypercall interface (nova.pdf §3-4: 'The microhypervisor implements a capability-based hypercall... the VMM cannot perform hypercalls without the capability'). TABOS adopts the KVM register convention to minimise the L2.3 device-model port.
- Guest GPRs are NOT swapped by the world switch — VM exit/entry only loads RSP/RIP/RFLAGS/segments/control-regs from the VMCS host/guest areas; RAX..R15 retain guest values that the host stub must explicitly save and later restore (SDM Vol 3C §27.5.1; already documented in the repo's world_switch.rs), which is why L2.2 must add a GprState and a save/restore asm stub before a multi-exit resume loop is correct.
