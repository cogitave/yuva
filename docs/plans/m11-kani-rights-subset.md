# Prove M11 capability rights-subset / no-confused-deputy with Kani (reused-code, host-verifiable tb-caps-core) plus an optional Verus unbounded model

> **Depends on:** none

> Auto-generated from the `tabos-forward-plans` ultracode planning workflow (research-backed, decision-closed). Treat as the working spec for this task; refine in place as implementation reveals detail.

## Goal
M11's capability system claims monotone attenuation: the only rights-changing primitive is bitwise-AND (Rights::intersect), so every derived/delegated capability's rights must be a SUBSET of its parent's, and no operation may widen authority (no confused deputy). Today this is asserted only by on-target boot smoke-tests (main.rs M11/M18 witnesses) and prose; M18's entire frozen-boundary safety guarantee is documented to "REDUCE TO the M11 rights-mask invariant", so it is load-bearing and must be a machine-checked proof, not a test. Done = a Kani harness suite, in-tree and CI-required, that (a) proves over the FULL u32 space that derive/narrow only ever shrinks rights (r.intersect(m) is a subset of r), and (b) proves that no sequence of {mint, dup, narrow, transfer, revoke} can make any resolvable handle's rights exceed its parent's (and that a forged handle yields no authority beyond its slot's). The proven algebra must be the SAME code the kernel runs (zero model drift), and a CI marker M11: caps-subset PROVEN must fail closed if any harness regresses or is silently deleted.

## Approach
1. 1. Scaffold the verification crate. Create crates/tb-caps-core (no_std, #![forbid(unsafe_code)], extern crate alloc) and add it to the workspace [workspace].members in Cargo.toml (before tb-hal). Add [lints.rust] unexpected_cfgs = { level = "warn", check-cfg = ['cfg(kani)'] } to its Cargo.toml (Kani usage.html guidance for Rust >=1.80 so cfg(kani) does not warn under the kernel's pinned nightly). This crate builds for the HOST triple under plain cargo (the repo deliberately keeps -Zbuild-std out of global .cargo/config.toml, bundling it only in the kbuild alias), so `cargo kani -p tb-caps-core` compiles cleanly without build-std interference.
2. 2. Lift the Rights algebra as the single source of truth. Move the Rights newtype and ALL its methods (bits/from_bits/union/intersect/contains/is_subset_of + the 13 named constants) verbatim from crates/tb-hal/src/caps.rs into crates/tb-caps-core/src/rights.rs; re-export `pub use rights::Rights;` from lib.rs. In tb-hal/src/caps.rs replace the inline definition with `pub use tb_caps_core::Rights;` and add tb-caps-core as a path dependency in crates/tb-hal/Cargo.toml. Now the kernel and Kani verify the EXACT SAME Rights code (zero drift).
3. 3. Extract the authority core as a generic CapTable<O>. Into crates/tb-caps-core/src/table.rs, factor the payload-agnostic logic out of tb-hal HandleTable: SlotCore<O>{generation:u32, rights:Rights, object:Option<O>}, fields slots:Vec<SlotCore<O>>/free:Vec<u32>/cap:usize, and methods with_capacity/alloc/live/free_slot/dup/narrow/revoke/transfer_to/detach/attach/rights_of (identical bodies, generic over O). Preserve the live() fail-closed order exactly: out-of-range -> BadCap, vacant -> Stale, generation mismatch -> Stale. Add a #[cfg(any(test, kani))] seed_slot(gen,rights,occupied) and peek_rights(slot) hook so harnesses can build arbitrary states.
4. 4. Re-wire tb-hal as a thin wrapper. Make tb-hal HandleTable hold `core: CapTable<Rc<Object>>`; the tb-hal-specific mint/mint_memory_home/mint_channel_endpoint/mint_block/mint_model_session helpers build the Rc<Object> then call core.alloc/attach. Keep the public tb-hal::caps API (Handle, HandleTable, dispatch, SyscallArgs, SysReturn, SysStatus, ObjKind) byte-identical so kernel/src/main.rs and the M11..M18 dispatch witnesses are untouched. Commit this as a PURE behavior-preserving refactor and prove it green (both arches build via cargo kbuild; QEMU boot still prints M11: caps OK through M18: evolve OK) BEFORE adding any harness.
5. 5. Tier-1 Rights proofs (complete, no unwind bound) in proofs.rs under #[cfg(kani)]: kani_intersect_subset_both (let r=Rights::from_bits(kani::any()); let m=...; assert r.intersect(m).is_subset_of(r) && .is_subset_of(m)); kani_intersect_monotone (a subset b => a.intersect(m) subset b.intersect(m)); kani_subset_reflexive_transitive (a subset b && b subset c => a subset c); kani_contains_subset_duality (r.contains(need) == need.is_subset_of(r)); kani_union_only_grows (r subset r.union(o); used only to BUILD, never on a live cap). Because each uses Rights::from_bits(kani::any::<u32>()), CBMC reasons over all 2^32 bit patterns symbolically -> a TOTAL proof of property (a).
6. 6. Tier-2 per-operation proofs on the REAL CapTable<()> (small cap N=4): kani_narrow_shrinks (mint root rights R=any; child=narrow(h,mask=any); assert rights_of(child) subset R); kani_dup_not_widen (rights_of(dup)==R, hence subset); kani_transfer_preserves_and_drains (transfer_to(dst): dst rights==R subset R AND src live(h)!=Ok -> no residual authority); kani_revoke_makes_stale (after revoke, live(old_h) is Stale and reissued slot's generation differs); kani_forged_handle_no_amplify (h=Handle::from_raw(kani::any::<u64>()); if live(h)==Ok(i) then resolved authority is EXACTLY slot[i].rights, never the forged value -> the unforgeability half of no-confused-deputy).
7. 7. Tier-3 headline inductive proof kani_step_preserves_attenuation on CapTable<()>: seed N=4 slots with nondeterministic state that respects the invariant BY CONSTRUCTION (each occupied slot's rights = ROOT.intersect(kani::any()) so rights subset ROOT, generation=any nonzero), kani::assume the table-level invariant, apply ONE nondeterministically-chosen op {dup, narrow(any mask), transfer round-trip, revoke} to a nondeterministically-chosen handle, then assert every occupied slot's rights still subset ROOT. A single-step preservation over arbitrary fixed-width state is an INDUCTION STEP -> the no-widen guarantee holds for sequences of UNBOUNDED length (bounded only in table width). Add kani_bounded_sequence (K<=4 concrete ops) as an independent cross-check.
8. 8. Add scripts/verify-caps.sh: run `cargo kani -p tb-caps-core --output-format=terse`, assert ZERO 'VERIFICATION:- FAILED' lines AND that the count of 'VERIFICATION:- SUCCESSFUL' equals a pinned EXPECTED_HARNESSES constant (fail closed on any mismatch, so a deleted/renamed harness can never pass silently), then echo the DoD marker 'M11: caps-subset PROVEN' and exit 0.
9. 9. Add .github/workflows/kani.yml (ubuntu-latest; Kani CI supports only x86_64-unknown-linux-gnu): step `uses: model-checking/kani-github-action@v<pinned>` with args `-p tb-caps-core --output-format=terse --sarif kani.sarif`, upload kani.sarif via github/codeql-action/upload-sarif@v3 (grant security-events: write), then run bash scripts/verify-caps.sh to emit/assert the marker. Mark this job a REQUIRED status check in branch protection (P0, load-bearing for M18).
10. 10. (Optional, non-blocking secondary) Add verify/tb-caps-verus/ OUTSIDE the kernel workspace (its own Verus toolchain): a small Verus model of the slot table with a spec-fn invariant (forall occupied slots: rights subset parent_rights subset root) and a proof-fn induction over derivation depth, giving the UNBOUNDED-width guarantee Kani's bounded checker cannot. Run it in a `verus-caps` job with continue-on-error: true so Verus toolchain churn never blocks the critical path.
11. 11. Documentation: in docs/MILESTONES.md record that M11's rights-subset/no-confused-deputy invariant is now machine-proven (Kani, marker M11: caps-subset PROVEN), in docs/BACKLOG.md tick task #3, and in docs/OPEN-QUESTIONS.md close the 'is the attenuation invariant proven or only tested' item.

## Decisions — no open questions (açık karar bırakma)
- Kani is the REQUIRED gate; Verus is OPTIONAL secondary. Kani verifies the REAL, unmodified reused Rust and ships an official GitHub Action (model-checking/kani-github-action); Verus would force the kernel build to carry vstd/builtin deps or maintain a hand-annotated dialect copy. So Kani proves the production code; Verus only proves a separate unbounded model in a non-blocking job.
- Separate verification crate (crates/tb-caps-core), NOT cfg(kani) harnesses inside tb-hal. tb-hal is the unsafe+assembly crate, and Kani emits assert(false)+assume(false) for asm! (unsupported), which would poison `cargo kani -p tb-hal`. Isolating the pure SAFE algebra into its own no_std+alloc crate keeps Kani's compile asm-free and makes the algebra the single source of truth.
- Factor BOTH the Rights algebra AND a generic authority core CapTable<O> into tb-caps-core; tb-hal HandleTable becomes a thin wrapper over CapTable<Rc<Object>>. This yields ZERO model drift: Kani verifies the exact slot/generation/free-list/rights code the kernel runs, with only the Object payload parameterized out.
- Verified instance is CapTable<()> (unit payload), not CapTable<Rc<Object>>. Per Kani feature support Vec/Rc are Yes but RefCell is only Partial; using () keeps the verified path to plain Vec<SlotCore<()>> with no Rc and no RefCell (the RefCell lives only in tb-hal's Object, which is outside the proof).
- Tier-1 Rights proofs construct values via Rights::from_bits(kani::any::<u32>()), covering all 2^32 patterns (not just the 13 named bits). This makes property (a) a COMPLETE bit-vector proof with no unwind bound.
- Table no-widen is proven by an INDUCTIVE single-step preservation harness over arbitrary fixed-width (N=4) seeded state, giving an unbounded-in-sequence guarantee, backed by per-op harnesses and a bounded K<=4 sequence cross-check. This is the standard way to lift a bounded model checker to an unbounded result and is honest about the residual bound (table width N=4).
- DoD marker is a CI marker 'M11: caps-subset PROVEN' (in-family with Mxx: ... OK), emitted by scripts/verify-caps.sh ONLY after asserting zero FAILED and SUCCESSFUL-count == pinned EXPECTED_HARNESSES. The on-target boot marker 'M11: caps OK' stays unchanged; this task adds a host PROOF, not a kernel behavior change.
- kani.yml runs ubuntu-latest only and is a REQUIRED, blocking check with SARIF uploaded to Code Scanning; the Kani action manages its own pinned toolchain (independent of rust-toolchain.toml), and we never invoke Kani through the kbuild alias, so build-std never contaminates the host verification.
- The extraction lands as a behavior-preserving refactor commit proven green (both arches boot M0..M18 unchanged) BEFORE harnesses are added, so any regression is attributable and the existing main.rs M11/M18 dispatch witnesses act as the refactor's regression net.

## Risks & mitigations
- Refactor breaks tb-hal behavior while extracting Rights + CapTable<O>. Mitigation: keep method bodies byte-identical (only generic over O), preserve live() fail-closed order (BadCap before Stale before generation), land it as a standalone refactor commit, and rely on the existing main.rs M11..M18 boot witnesses + dispatch tests as the regression net before any harness work.
- Kani's pinned toolchain conflicts with the repo's rust-toolchain.toml nightly. Mitigation: use the official model-checking/kani-github-action (it sets its own RUSTUP_TOOLCHAIN internally), keep tb-caps-core plain no_std+alloc with no exotic features, add the check-cfg lint for cfg(kani), and never run cargo kani through the kbuild alias (which would force -Zbuild-std).
- Bounded-model-checking soundness gap: table proofs are bounded to cap N=4. Mitigation: Tier-1 algebra proof is COMPLETE over the full u32 space; Tier-3 is an inductive single-step over arbitrary seeded state (unbounded in sequence length); the optional Verus model closes the unbounded-width gap by induction; the N=4 bound is documented in each harness.
- alloc/Rc/Vec/RefCell support in Kani is uneven (RefCell only Partial). Mitigation: verify CapTable<()> so the proven path uses only Vec<SlotCore<()>> (no Rc, no RefCell); the RefCell-bearing Object payload is entirely outside the verified instance.
- CBMC state-space blowup on the sequence/inductive harnesses inflates CI time or exhausts the ~7GB/6h GitHub runner budget. Mitigation: small cap (N=4) and short sequences (K<=4), --output-format=terse, per-harness #[kani::solver]/#[kani::unwind] tuning, and the option to move the heaviest harness to a scheduled (cron) run while keeping the fast Tier-1/Tier-2 set in the required PR gate.
- A green job with zero (or vacuous) harnesses gives false assurance. Mitigation: verify-caps.sh pins EXPECTED_HARNESSES and fails closed on any count mismatch, and the negative-control check (AND->OR must FAIL) is documented to keep the harnesses non-vacuous.

## Research notes (with sources)
- Kani is a bounded model checker built on CBMC: #[kani::proof] harnesses use kani::any::<T>() to stand for all values of T and kani::assume() to set preconditions; by default it finds reachable panics/assert failures/overflow/UB without explicit asserts (source: Kani tutorial-first-steps.html).
- Kani sets cfg(kani) on the target crate, injects the kani crate, and sets cfg(kani_host) for host build scripts; harnesses MUST be wrapped in #[cfg(kani)] so normal cargo build is unaffected, and Rust >=1.80 wants [lints.rust] unexpected_cfgs check-cfg=['cfg(kani)'] in Cargo.toml (source: Kani usage.html).
- Kani Rust-feature support: Structs/Enums/Unions/Traits/Generics/Box/Rc/Arc are Yes; RefCell/UnsafeCell/PhantomData/Drop are Partial; assembly is NOT supported (asm! compiles to assert(false)+assume(false)) and concurrency is out of scope. A pure SAFE, single-threaded, asm-free rights algebra over Vec<u32-state> sits squarely in the supported subset (source: Kani rust-feature-support.html).
- Kani ships an official GitHub Action model-checking/kani-github-action@vMAJOR.MINOR that runs cargo kani on ubuntu (only x86_64-unknown-linux-gnu supported in CI), supports --output-format=terse and --sarif <path> for GitHub Code Scanning, and is installed out-of-band via cargo install --locked kani-verifier + cargo kani setup, which downloads Kani's OWN pinned toolchain independent of rust-toolchain.toml; GitHub runners cap ~7GB RAM / 6h (source: Kani install-github-ci.html).
- Kani has -Z concrete-playback (--concrete-playback=print|inplace) that emits a Rust unit test reproducing a failing harness with the concrete counterexample bytes - the mechanism to surface a real confused-deputy counterexample if a harness fails (source: Kani concrete-playback.html).
- Verus is an SMT/Z3 deductive verifier that statically proves full functional correctness with no runtime checks, but requires code written in its dialect (requires/ensures/forall/exists, spec/proof fns, int/nat) and explicitly does NOT aim to support arbitrary Rust/libraries; induction is done via recursive proof fns. This makes it complementary to (not a drop-in replacement for) Kani's bounded checking of unmodified Rust (source: Verus guide overview).
- Repo ground truth: crates/tb-hal/src/caps.rs is #![forbid(unsafe_code)]; Rights is #[repr(transparent)] u32 with intersect()=AND (the ONLY narrowing primitive), union()=OR (build-only), is_subset_of()/contains(); HandleTable exposes mint/dup(same rights)/narrow(old&mask)/transfer_to(move, same rights)/revoke(bump generation)/detach/attach, and live() resolves out-of-range->BadCap, vacant->Stale, generation-mismatch->Stale. There are currently NO #[cfg(test)] or Kani harnesses in caps.rs (source: tabos/crates/tb-hal/src/caps.rs).
- The on-target M11 DoD marker is the serial line 'M11: caps OK' (kernel/src/main.rs ~line 1230); the M18 self-test comment states the frozen-kernel guarantee 'REDUCES TO the M11 rights-mask invariant' and prints 'M18: evolve OK', confirming this proof is load-bearing for M18 (source: tabos/kernel/src/main.rs lines 1090-1230 and 2213-2414).
- Repo build/CI facts that make a host crate verifiable cleanly: .cargo/config.toml deliberately keeps -Zbuild-std OUT of global config (it is bundled ONLY in the `kbuild` alias) so plain cargo / cargo kani on the host triple work for non-kernel crates; workspace members are crates/tb-boot, crates/tb-hal, kernel (tb-vmm is an excluded separate workspace); ci.yml installs nightly via dtolnay/rust-toolchain@nightly and boots both arches under pure-TCG QEMU, grepping serial for the DoD marker (source: tabos/.cargo/config.toml, tabos/Cargo.toml, tabos/.github/workflows/ci.yml).
- The caps.rs design note states the model is validated against the seL4 Capability Derivation Tree (CDT) and Zircon handles, where authority lives in the SLOT not the handle and rights only ever attenuate along derivation; the no-amplification / no-confused-deputy property this task formalizes is the standard capability-system invariant that seL4's CDT and Zircon's rights masks enforce (source: in-tree design doc comment in tabos/crates/tb-hal/src/caps.rs; external seL4/Zircon pages were not fetchable in this run because the web-search backend lacked an API key).
