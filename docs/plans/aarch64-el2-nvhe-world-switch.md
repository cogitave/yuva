# aarch64 EL2 (nVHE) world-switch smoke proof — "L2.0: el2 OK" under pure-TCG QEMU virt

> **Task #48** · DoD marker: `L2.0: el2 OK` · Effort: **L** · Slug: `aarch64-el2-nvhe-world-switch`
> **Depends on:** #3, #8

> Auto-generated from the `tabos-forward-plans` ultracode planning workflow (research-backed, decision-closed). Treat as the working spec for this task; refine in place as implementation reveals detail.

## Goal
Prove the EL2 sovereignty primitive on ARM that VMX-root proves on x86: TABOS boots at EL2 (QEMU `virt,virtualization=on`), installs a resident nVHE EL2 monitor (VBAR_EL2 + HCR_EL2.RW), drops to EL1 to run the existing M0..M18 chain unchanged, then at the final L2 milestone slot does a real EL2 world-switch — ERET into a tiny EL1 guest stub that executes `HVC #imm` and traps back to EL2 — caught and verified. Confined entirely to tb-hal (the kernel crate stays `#![forbid(unsafe_code)]`), driven by a safe `el2_selftest() -> El2Proof` facade mirroring the existing `vmx_selftest()/VmxProof` pattern. DONE = the aarch64 boot prints `L2.0: el2 OK` after a genuine ERET→guest→HVC→EL2 round-trip with the guest's magic captured, while every prior marker M0..M18 + the x86-only vmxroot n/a line still print; CI greps `el2 OK`. Unlike L2.0 vmxroot (which only skips under TCG), this proof actually EXECUTES under pure TCG on a stock hosted runner.

## Approach
1. BOOT: rewrite `_start` in crates/tb-hal/src/arch/aarch64/boot.rs to be EL-aware. At entry: `mov x19,x0` (stash FDT), `mov x20,#0` (default not-EL2). Read `mrs x1,CurrentEL; lsr x1,#2; cmp x1,#2`; if not EL2, `b.ne` straight to the EL1 continuation label (legacy direct-EL1 boot path, x20 stays 0). x0,x1,x2 are scratch from here (FDT safe in x19).
2. EL2 nVHE setup (only when entered at EL2): program the minimal subset of Linux's `init_el2_state` for cortex-a72/GICv2/softfloat — `SCTLR_EL2 = 0x30C50830` (INIT_SCTLR_EL2_MMU_OFF, RES1 bits, MMU off), `HCR_EL2 = 1<<31` (RW=1 -> EL1 is AArch64; VM=0 stage-2 OFF, TGE=0, IMO/FMO/AMO=0 so IRQs stay at EL1, HCD=0 so HVC is enabled), `CNTHCTL_EL2 = 0x3` (EL1PCTEN|EL1PCEN -> EL1 physical timer/counter NOT trapped, required for M8) and `CNTVOFF_EL2 = 0`, `CPTR_EL2 = 0x33FF` (no FP/copro traps to EL2), `HSTR_EL2 = 0`, `VTTBR_EL2 = 0`, `VPIDR_EL2 = MIDR_EL1`/`VMPIDR_EL2 = MPIDR_EL1`, `VBAR_EL2 = __el2_exception_vectors`, `SP_EL2 = __el2_stack_top`. Set `x20=1`. Then `ELR_EL2 = 1f`, `SPSR_EL2 = 0x3C5` (EL1h + DAIF masked = INIT_PSTATE_EL1), `isb`, `eret` -> drops to EL1 at label 1f.
3. EL1 continuation (label 1f, shared by both paths): set `SP=__boot_stack_top`, zero `.bss [__bss_start,__bss_end)`, then (AFTER bss-zero, so it survives) `adrp x1,BOOTED_AT_EL2; strb w20,[x1]` to record the entry-EL flag; arm `VBAR_EL1=__exception_vectors`+isb; `mov x0,x19` (restore FDT as arg0); `b rust_main`. M0..M18 then run at EL1 byte-for-byte as today.
4. EL2 VECTORS: create crates/tb-hal/src/arch/aarch64/el2_vectors.rs as a pure `global_asm!` module (mirror vectors.rs) — a 2 KiB-aligned `.text._el2_vectors` table of 16x128B slots. EL2 variants of SAVE_CONTEXT/RESTORE_CONTEXT push/pop x0..x30 + ELR_EL2 + SPSR_EL2 into a 0x110-byte frame on SP_EL2 (identical layout to the EL1 TrapFrame but using the *_EL2 system registers). The 'Lower EL using AArch64, Synchronous' slot (index 8, offset 0x400) -> `__el2_vec_lower_sync`: SAVE_CONTEXT, `mov x0,sp`, `bl aarch64_el2_sync_handler` (the handler erets itself, like M4's __vec_el0_sync). All 15 other slots -> `__el2_vec_other`: a fatal stub that records an unexpected-EL2 marker and erets/halts.
5. EL2 HANDLER + driver: create crates/tb-hal/src/arch/aarch64/el2.rs. Define `#[no_mangle] static BOOTED_AT_EL2: AtomicU8` (written by `_start`). `aarch64_el2_sync_handler(frame: *mut Frame)` (extern "C", runs at EL2, MMU-off): read `ESR_EL2`, assert EC==0x16 (HVC64). Dispatch on `ESR_EL2.ISS[15:0]`: imm==0 (the kernel BOOTSTRAP hvc) -> leave frame B0 on the EL2 stack untouched, set `ELR_EL2=guest_stub` (its identity VA==PA), `SPSR_EL2=0x3C5`, `isb`, `eret` INTO the tiny guest at EL1. imm==1 (the GUEST done-hvc) -> read `frame.gpr[0]` (the guest magic 0xE12), set outcome (0 ok / nonzero fail), compute `b0 = frame + 0x110` (B0 sits exactly one frame above B1 because B0 was never popped), write `b0.gpr[0]=outcome`/`b0.gpr[1]=magic`, then asm `mov sp, b0_base; RESTORE_CONTEXT` (pops B0: restores x0..x30 + ELR_EL2 + SPSR_EL2) + `eret` -> returns to the kernel's post-hvc PC with x0=outcome and all kernel regs transparent.
6. GUEST STUB: a position-independent EL1 stub (`#[unsafe(naked)]`/global_asm, like user_stub): `mov x0,#0xE12; hvc #1; 1: b 1b`. It runs at EL1h under the live kernel stage-1 MMU (HCR_EL2.VM=0, no stage-2), uses no stack (SP_EL1 untouched), and its only effect is the trap-back. Mirrors the x86 single-CPUID guest and the M4 EL0 svc stub.
7. FACADE: `el2_selftest() -> El2Proof` in el2.rs. If `BOOTED_AT_EL2 != 1` return `El2Proof::Unavailable` (graceful skip, no HVC issued -> no fault). Else: mask EL1 IRQs (belt-and-suspenders, mirrors vmx_selftest's `cli`), issue `asm!("hvc #0", out("x0") code, clobber_abi("C"))`, restore IRQs; map code 0 -> `Proven{hvc_imm:1}`, nonzero -> `RoundTripFailed{code}`. All asm/unsafe lives here.
8. LIB FACADE: in crates/tb-hal/src/lib.rs add `pub enum El2Proof { NotApplicable, Unavailable, RoundTripFailed{code:u64}, Proven{hvc_imm:u64} }` plus `#[cfg(target_arch="aarch64")] pub fn el2_selftest()->El2Proof { arch::el2_selftest() }` and `#[cfg(target_arch="x86_64")] pub fn el2_selftest()->El2Proof { El2Proof::NotApplicable }` — exactly mirroring the existing VmxProof/vmx_selftest block. Wire `mod el2; mod el2_vectors;` + `pub use el2::el2_selftest;` into arch/aarch64/mod.rs.
9. KERNEL MARKER: in kernel/src/main.rs, immediately AFTER the existing L2.0 vmx `match` block and before `tb_hal::halt()`, add a `match tb_hal::el2_selftest()` block: `Proven{..}` -> `serial_write_str("L2.0: el2 OK
")`; `Unavailable` -> `"L2.0: el2 OK (no EL2, skipped)
"`; `NotApplicable` -> `"L2.0: el2 OK (aarch64-only, n/a on x86_64)
"`; `RoundTripFailed{code}` -> print `"L2.0: el2 FAIL code="` + hex (no 'el2 OK' substring -> red). Kernel crate stays unsafe-free.
10. LINKER: in kernel/linker/aarch64.ld add `KEEP(*(.text._el2_vectors))` to the `.text` block right after the existing `.text._vectors` KEEP; add a dedicated 16 KiB EL2 stack to the `.bss` block (`__el2_stack_bottom`/`. = . + 0x4000`/`__el2_stack_top`) mirroring `__boot_stack_top` — a region the EL1 kernel never references (single-accessor for the MMU-off EL2 handler).
11. RUN SCRIPT + CI: in scripts/run-aarch64.sh change the QEMU line to `-M virt,virtualization=on,gic-version=2 -cpu cortex-a72` (boots at EL2; GICv2 keeps M8's hard-coded addresses valid) and set `MARKER="el2 OK"`. No .github/workflows/ci.yml edit needed (the aarch64 job just calls the script; `qemu-system-arm` package provides qemu-system-aarch64).
12. DOC: append an implementation-status note to docs/SOVEREIGNTY-L2-ROADMAP.md §5/§9 recording the ARM EL2 smoke as landed and that it executes for real under pure TCG (the one L2 rung whose world-switch is NOT a CI skip). Add Tier-1 `const _: () = assert!(...)` checks in el2.rs locking the constants (SPSR=0x3C5, CNTHCTL=0x3, SCTLR_EL2=0x30C50830, EC_HVC64=0x16, frame size 0x110).

## Decisions — no open questions (açık karar bırakma)
- RUN AT THE MILESTONE SLOT, NOT AT BOOT (resident EL2 monitor): chose Plan A — keep a dormant nVHE EL2 monitor resident across the whole boot and trigger the proof at the final L2 slot via a bootstrap `HVC #0` from the running EL1 kernel — over Plan B (run the proof in `_start` before the drop). Rationale: faithful to the mature nVHE split-VMM (EL2 entered by HVC from a live guest = the hypercall primitive), serial is up so it reports at the right slot, and it additionally exercises the real EL1->EL2 HVC path. Mirrors how L2.0 vmxroot runs as an end-of-boot self-test inside the booted kernel.
- BOOT AT EL2 + DROP TO EL1 for the whole chain: enable `virtualization=on` (entry becomes EL2) and immediately drop to EL1 via the standard Linux nVHE recipe, so M0..M18 keep running at EL1 unchanged. Running M0..M18 at EL2 was rejected (it would break M3's SCTLR_EL1-based MMU test and the M4 EL0 excursion, since the kernel's translation/EL transitions are EL1-shaped).
- STAGE-2 DISABLED for the smoke: HCR_EL2.VM=0, VTTBR_EL2=0, VTCR_EL2 left unprogrammed. The task allows 'stage-2 disabled-or-identity'; VM=0 is simplest and the guest stub runs correctly under the kernel's live EL1 stage-1 (identity low-RAM). A stage-2 IPA->PA identity builder + TLBI VMALLS12E1 (backlog #18's stage2.rs) is DEFERRED to the follow-on ARM rung — it belongs to the 'el2 ept'-equivalent, not the world-switch smoke.
- MARKER = `L2.0: el2 OK`: this is the aarch64 realization of the same L2.0 'we are the hypervisor' rung that vmxroot realizes on x86. Both arches print both lines each boot — the irrelevant one as a green n/a/skip (`L2.0: vmxroot OK (x86-only, n/a on aarch64)` and, on x86, `L2.0: el2 OK (aarch64-only, n/a on x86_64)`). Distinct grep substrings ('vmxroot OK' vs 'el2 OK') keep the two L2.0 lines unambiguous; each run script greps its own arch's real proof.
- CPU = cortex-a72 (NOT `-cpu max` as backlog #18 suggested): a72 is pure ARMv8.0 (no VHE -> guaranteed nVHE, HCR_EL2.E2H is RES0), already validated for M0..M18, and defaults to GICv2; `-cpu max` would pull in VHE/GICv4 surface and risk M8 regressions. Pin `gic-version=2` explicitly so M8's hard-coded GICv2 GICD/GICC MMIO addresses stay correct.
- RESULT VIA REGISTER, NOT A CACHEABLE STATIC: the proof outcome returns in x0 by overwriting B0.gpr[0] before RESTORE_CONTEXT, avoiding the EL2-MMU-off (Device/non-cacheable) vs EL1-Normal-WB cache-coherency hazard. The only cacheable static touched cross-EL is BOOTED_AT_EL2, written once at boot with caches off and read later via a cold fill (coherent). The EL2 stack and saved frames are a dedicated region the EL1 kernel never references (single-accessor invariant).
- HVC IMMEDIATE DISAMBIGUATION via ESR_EL2.ISS: bootstrap kernel uses `hvc #0`, guest uses `hvc #1`; the shared EL2 'Lower-EL AArch64 Synchronous' (0x400) handler branches on ISS[15:0] after asserting EC==0x16. B0 (bootstrap frame) is recovered deterministically as B1+0x110 because the imm==0 path erets to the guest without popping B0 — no auxiliary static needed.
- x0=FDT PRESERVED across the drop: stashed in callee-saved x19 at `_start` entry and restored to x0 at the EL1 continuation, so the existing FDT/DTB handoff (M10 pmm, etc.) is byte-identical. The entry-EL flag is threaded in x20 and stored into BOOTED_AT_EL2 only AFTER bss-zero so it is not wiped.
- FACADE SHAPE mirrors VmxProof exactly: El2Proof{NotApplicable(x86)|Unavailable(no EL2 -> green skip)|RoundTripFailed{code}(booted-EL2 but failed -> honest red)|Proven{hvc_imm}}. All silicon-unsafe/asm confined to tb-hal/arch/aarch64/{el2,el2_vectors}.rs; the kernel only branches on the enum, preserving the framekernel invariant.
- DECOUPLE FROM THE aarch64 tb-boot PRODUCER: backlog #18 lists 'Build aarch64 tb-boot producer + _tb_start EL1 entry' as a predecessor, but that is only needed to run the REAL TABOS kernel as the EL1 guest (a later rung). The smoke uses a tiny in-tree stub guest, so the producer is NOT required — letting `el2 OK` land NOW on stock TCG (the roadmap §9 ARM parallel smoke-track).

## Risks & mitigations
- Enabling virtualization=on flips the entry EL to EL2, so a wrong EL2->EL1 drop regresses ALL of M0..M18. MITIGATION: implement the exact Linux nVHE recipe (cited el2_setup.h/sysreg.h), preserve x0=FDT via x19, leave SCTLR_EL1 at QEMU reset (identical to a direct-EL1 boot), and gate merge on the full M0..M18 chain still printing under the new boot.
- M8 timer regression: the EL1 physical timer (CNTP, PPI 30) traps to EL2 unless CNTHCTL_EL2.EL1PCEN/EL1PCTEN are set, or IRQ routing changes. MITIGATION: set CNTHCTL_EL2=0x3 + CNTVOFF_EL2=0, keep HCR_EL2.IMO/FMO/AMO=0 so physical IRQs stay at EL1 (VBAR_EL1), and pin gic-version=2 so the GICv2 distributor/CPU-interface addresses M8 hard-codes remain valid.
- EL2-MMU-off cache aliasing: the EL2 handler runs with SCTLR_EL2.M=0 (accesses are Device/non-cacheable) while EL1 maps the same RAM Normal-WB cacheable. MITIGATION: dedicate `__el2_stack` (EL1 never touches it -> single-accessor), pass the result in a register not a cacheable static, and keep BOOTED_AT_EL2 a boot-time (caches-off) write read later via a cold fill. Documented as an invariant; a full EL2 MMU is deferred to a later rung.
- Infinite EL2 re-entry if the bootstrap vs guest HVC are confused, or if a non-HVC EL2 exception is mis-handled. MITIGATION: assert ESR_EL2.EC==0x16 and branch strictly on ISS (#0 vs #1); route all 15 other EL2 vector slots to a fatal `__el2_vec_other` stub that prints a diagnostic and halts (never silently loops).
- QEMU/runner variance: a runner whose QEMU lacks EL2 for cortex-a72 under TCG would not boot at EL2. MITIGATION: the standard lane pins virtualization=on (validated locally on QEMU 6.2 and on stock hosted runners per roadmap §9); if EL2 is truly absent the BOOTED_AT_EL2 probe yields a green skip rather than a fault, and the failure is visible (no real `el2 OK`).
- Guest stub un-executable at EL1 (wrong permissions/PXN) would data/instruction-abort instead of HVC-ing. MITIGATION: place the stub in identity-mapped kernel `.text` (already EL1-executable, the kernel runs from there) and ERET to its VA==PA; the abort path is caught by the EL2 'other' vector and surfaces as a FAIL marker, not a hang.
- Two `L2.0:` lines per boot could confuse marker tooling/benches (bench-boot.sh keys on `L2.0: vmxroot OK`). MITIGATION: distinct substrings ('vmxroot OK' vs 'el2 OK'); the aarch64 script greps `el2 OK`, x86 keeps `vmxroot OK`; bench-boot.sh (x86 path) is untouched.
- AAPCS clobber correctness across `hvc #0`: if the EL2 round-trip fails to restore the kernel's registers, the post-HVC kernel code mis-behaves. MITIGATION: the imm==1 path does a full RESTORE_CONTEXT of B0 (x0..x30 + ELR_EL2 + SPSR_EL2), making the bootstrap HVC fully transparent except for x0=result; the facade additionally declares clobber_abi("C").

## Research notes (with sources)
- QEMU `virt` machine: the `virtualization` option 'Set on/off to enable/disable emulating a guest CPU which implements the Arm Virtualization Extensions. The default is off.' -> `virtualization=on` is what provides EL2 to the guest; cortex-a72 is a supported 64-bit CPU; RAM starts at 0x4000_0000. (QEMU docs, system/arm/virt.html, fetched 2026-06-08.)
- Roadmap §5/§9 (SOVEREIGNTY-L2-ROADMAP.md): 'QEMU virt -machine virtualization=on -cpu cortex-a57|max provides EL2 to the guest under PURE TCG — NO nested KVM, NO /dev/kvm — so an aarch64 EL2 world-switch proof ("el2 OK") runs on ANY stock GitHub-hosted runner.' This is the unique CI advantage vs the x86 L2.0 lane (which TCG refuses VMX for, so it only skips).
- nVHE timer enable (Linux v6.6 arch/arm64/include/asm/el2_setup.h `__init_el2_timers`): for the non-VHE path CNTHCTL_EL2 = 3 (EL1PCTEN|EL1PCEN, 'Enable EL1 physical timers') and `msr cntvoff_el2, xzr`. Required so M8's EL1 physical timer/counter are not trapped to EL2.
- SCTLR RES1 values (Linux v6.6 arch/arm64/include/asm/sysreg.h): INIT_SCTLR_EL2_MMU_OFF = SCTLR_EL2_RES1 = bits{4,5,11,16,18,22,23,28,29} = 0x30C50830 (EL2 runs MMU-off with RES1 set); INIT_SCTLR_EL1_MMU_OFF RES1 bits{LSMAOE29,nTLSMD28,EIS22,TSCXT20,EOS11}=0x30500800 (cited; not written by the plan since QEMU's EL1 reset value already matches a direct-EL1 boot).
- nVHE eret PSTATE (Linux el2_setup.h `__init_el2_nvhe_prepare_eret`): SPSR_EL2 = INIT_PSTATE_EL1 = EL1h + D|A|I|F masked = 0x3C5 — identical to the value the existing TABOS aarch64 boot.rs already documents (PSTATE=0x3c5=EL1h, DAIF masked) and the M4 enter_el0 uses 0x3C0 for EL0t.
- HCR_EL2 fields (Arm ARM DDI 0487): RW = bit 31 (1 => the next lower EL, EL1, is AArch64; reset 0 => AArch32, so RW MUST be set); VM = bit 0 (1 => stage-2 enabled for EL1&0; 0 => stage-2 disabled, chosen for the smoke); HCD = bit 29 (HVC disable; reset 0 => HVC from EL1 is enabled and traps to EL2). The plan writes HCR_EL2 = 1<<31 (RW only).
- ESR EC for HVC (Linux arch/arm64/include/asm/esr.h): ESR_ELx_EC_HVC64 = 0x16 (HVC executed in AArch64 state); EC field is bits[31:26] (shift 26, mask 0x3F) — same handling the codebase already uses for ESR_EL1 (SVC64=0x15 in user.rs, BRK64=0x3C in trap.rs). The EL2 handler reads ESR_EL2 (sys_reg 3,4,5,2,0 per sysreg.h).
- VBAR_EL2 layout: a 2 KiB-aligned table of 16 x 128-byte slots, same structure as VBAR_EL1; an HVC from a lower EL (EL1) in AArch64 state is taken through the 'Lower EL using AArch64, Synchronous' slot at offset 0x400 (index 8) — confirmed by the existing TABOS vectors.rs which uses offset 0x400 for the EL0 synchronous (svc) slot. (Arm ARM D1 exception vectors; SYS_VBAR_EL2 = sys_reg(3,4,12,0,0).)
- Codebase patterns reused verbatim: the M4 user.rs enter_el0 (program ELR/SPSR + eret to a lower EL) and its setjmp/longjmp handler that does NOT return to the vector trampoline; vectors.rs SAVE_CONTEXT/RESTORE_CONTEXT 0x110-byte frame (x0..x30 + ELR + SPSR); trap.rs ESR-EC dispatch; and the lib.rs VmxProof/vmx_selftest facade (NotApplicable on the other arch, kernel branches on the enum). The EL2 work is these lifted 'one EL up' with the *_EL2 registers.
- x86 analogue (world_switch.rs/vmx/mod.rs): VMX-root is entered from ring 0 at will, but EL2 on ARM is a HIGHER EL than EL1 and cannot be self-raised — the only entries are boot-at-EL2 or an exception (HVC) routed to a resident EL2 monitor. This asymmetry is why the proof must boot at EL2, stay resident, and be re-entered via HVC, rather than 'turn on' EL2 mid-kernel like VMXON.
- Backlog #18 (docs/BACKLOG.md) is the canonical entry for this task ('Implement aarch64 EL2 (nVHE) world-switch smoke proof (el2 OK) under TCG'); it lists stage2.rs/VTCR_EL2/VTTBR_EL2 and `-cpu max`, both of which this plan deliberately narrows (stage-2 disabled for the smoke; cortex-a72+gic-version=2 for M8 parity) with explicit rationale.
