# M14 byte-payload IPC: copy_to_user/copy_from_user bounce-buffer round-trip across two agent address spaces

> **Task #50** · DoD marker: `M14.1: payload OK` · Effort: **L** · Slug: `m14-byte-payload-ipc`
> **Depends on:** none

> Auto-generated from the `tabos-forward-plans` ultracode planning workflow (research-backed, decision-closed). Treat as the working spec for this task; refine in place as implementation reveals detail.

## Goal
Extend the complete-but-control-word M14 channel so a message can also carry a variable-length BYTE payload that is safely copied across two agents' isolated address spaces, lifting the one deliberately-deferred M14 unsafe. Today an `ipc::Message` is `{ payload: u64, cap: Option<(Rc<Object>,Rights)> }`; the byte payload must travel as a kernel-owned bounce buffer filled by a sender-side `copy_from_user` and drained by a receiver-side `copy_to_user`, with the two raw copy primitives confined to NEW per-arch `tb-hal` modules (the kernel crate stays `#![forbid(unsafe_code)]` and only orchestrates). Done = a real, distinct-PA byte buffer minted in agent_c's address space round-trips byte-for-byte into agent_d's address space through the channel FIFO, every copy is bounds- and permission-checked and fail-closed, and the boot self-test prints a greppable proof marker, re-run every boot on both x86_64 (QEMU microvm PVH + tb-vmm) and aarch64 (QEMU virt).

## Approach
1. 1. Add the typed fault + the two copy primitive SIGNATURES to the arch facade. In crates/tb-hal/src/arch/mod.rs declare a pure-data `pub enum CopyFault { NotPresent, NotUser, NotWritable, NotInPhysmap }` (Copy, Eq) and re-export `copy_to_user`/`copy_from_user` from the active arch. Signatures (identical both arches): `pub fn copy_from_user(root_pa: u64, src_uva: u64, dst: &mut [u8]) -> Result<(), CopyFault>` and `pub fn copy_to_user(root_pa: u64, dst_uva: u64, src: &[u8]) -> Result<(), CopyFault>`. The `root_pa` is passed EXPLICITLY (the target agent's top-level table PA from `agent_root_pa`), so the copy never depends on which root is live in CR3/TTBR0 and never switches it.
2. 2. Implement crates/tb-hal/src/arch/x86_64/uaccess.rs (the ONLY new x86 unsafe). Private `fn walk_user(root_pa, uva, write) -> Result<u64, CopyFault>`: software-walk PML4->PDPT->PD->PT starting at `root_pa` (identity-mapped low RAM, PA==VA, `read_volatile` each entry); at every level require `PTE_P` (else NotPresent) and `PTE_US` bit2 (else NotUser; Intel SDM Vol.3A 4.6 demands U/S=1 at EVERY level); reject any `PTE_PS` huge entry in the user range as NotPresent (TABOS maps only 4 KiB user leaves -> fail-closed, never misinterpret); on the leaf require `PTE_RW` when `write` (else NotWritable); return `(leaf_pa & ADDR_MASK) | (uva & 0xFFF)`. Then a public `fn copy_span(root_pa, uva, buf, write, to_user)` that loops PAGE-BY-PAGE (a buffer at an unaligned UVA spans >=2 pages, mirroring rust-vmm's fragmented `get_slices`): for each page compute the in-page length, `walk_user` it, assert the translated PA lies inside the boot identity window [0,1 GiB) (else NotInPhysmap), and `copy_nonoverlapping` that slice between `buf` and the PA's identity alias. copy_to_user/copy_from_user are thin wrappers (write=true/to_user vs write=false/from_user). Copy is BYTE-granular (`*const u8`/`*mut u8`) so an arbitrary user VA never triggers an unaligned u64 access (Linux unaligned-memory-access doc). Access goes through the kernel SUPERVISOR identity alias, NOT the user VA, so SMAP/PAN can never fire (matches the existing M4 `map_user_pages` note: writing through a frame's identity address is SMAP-immune).
3. 3. Implement crates/tb-hal/src/arch/aarch64/uaccess.rs symmetrically (the ONLY new aarch64 unsafe). Walk L1->L2->L3 from `root_pa`; at each table level require descriptor low bits == 0b11 (valid table) else NotPresent; on the L3 leaf require page desc 0b11 + `DESC_AF` (bit10) + `AP_EL0` (bit6, EL0-accessible) else NotUser, and for `write` require `AP_RDONLY` (bit7) CLEAR else NotWritable (Arm ARM VMSAv8-64 AP[2:1]=bits[7:6], transcribed in aarch64/mmu.rs). Assert the leaf PA is inside the identity-mapped RAM gigabyte [1 GiB,2 GiB) (QEMU virt RAM base) else NotInPhysmap; access via PA==VA identity alias. Register `mod uaccess; pub use uaccess::{copy_to_user, copy_from_user};` in both arch/{x86_64,aarch64}/mod.rs.
4. 4. Extend crates/tb-hal/src/ipc.rs (stays `#![forbid(unsafe_code)]`). Add `bytes: Option<alloc::boxed::Box<[u8]>>` to `Message` (the bounce buffer, kernel-heap-backed; `None` = no byte payload, the existing inline-scalar path). Add `pub const MAX_PAYLOAD: usize = 4096;`. Add a `Message::byte_len(&self) -> usize` peek helper for the receiver's too-small-buffer check. No new unsafe -- a `Box<[u8]>` is safe heap, the same soundness argument the existing `Rc`/`RefCell` rings already discharge.
5. 5. Extend crates/tb-hal/src/caps.rs (stays `#![forbid(unsafe_code)]`). Add `SysStatus::Fault = 9` AFTER `PeerClosed = 8` (keep the closed status surface total) documenting it as a user-buffer copy fault / precondition miss -- nothing enqueued or dequeued. Add root-carrying bodies the kernel facade calls: `pub(crate) fn chan_send_bytes(&mut self, ep, payload, cap_raw, src_root, src_uva, len) -> SysStatus` and `pub(crate) fn chan_recv_bytes(&mut self, ep, dst_root, dst_uva, dst_cap) -> (SysStatus, u64, u64, usize)` (status, payload, moved-handle-raw, bytes-len). SEND order for atomicity: WRITE-gate -> peer_open -> outbox-space -> `len <= MAX_PAYLOAD` (else Denied) -> `copy_from_user(src_root, src_uva, &mut box[..len])` (on CopyFault return Fault, NOTHING detached/enqueued) -> ONLY THEN detach the optional carried cap (TRANSFER-gated, self-channel-rejected, as today) -> `push_back(Message{payload, bytes:Some(box), cap})`. RECV order: READ-gate -> peek head `byte_len`; if `dst_cap < byte_len` return Fault WITHOUT popping (Zircon `zx_channel_read` BUFFER_TOO_SMALL-without-discard: the message stays for a retry with a bigger buffer) -> pop -> if it has bytes call `copy_to_user(dst_root, dst_uva, &bytes)`; on CopyFault `push_front` the WHOLE message back (restores FIFO + parked cap + bytes, zero loss) and return Fault -> else `attach` the carried cap (as today) and return Ok. Plumb `M_CHAN_SEND`/`M_CHAN_RECV` in `dispatch` so the real user-syscall path (which has the current agent's root via `agent_syscall_current`) reads `args[2]=uva, args[3]=len/dst_cap` -- the roadmap's reserved `args[2..3]`.
6. 6. Add the kernel-side ORCHESTRATION facades to crates/tb-hal/src/lib.rs (safe; they call the unsafe arch primitives through caps bodies): `agent_map_user_buffer(t: Task) -> Option<u64>` (frame_alloc a fresh page, `arch::map_user_in_root(root, AGENT_BUF_VA, pa, writable=true, exec=false)`, return AGENT_BUF_VA); `agent_chan_send_bytes(t, ep, payload, cap_h, src_uva, len) -> Option<(u32,u64)>` (resolves `agent_root_pa(t)`, calls `chan_send_bytes`); `agent_chan_recv_bytes(t, ep, dst_uva, dst_cap) -> Option<(u32,u64,u64,usize)>` (resolves root, calls `chan_recv_bytes`). These mirror M15's `agent_block_map` precedent exactly: an address-space-dependent op rides the kernel facade because `caps::dispatch` alone cannot resolve an agent root.
7. 7. Wire the self-test into the existing M14 block in kernel/src/main.rs (after the current control-word assertions, before `M14: ipc OK`). Map a writable user buffer into BOTH agent_c and agent_d (`agent_map_user_buffer`); seed agent_c's buffer with a 1024-byte known pattern by writing through the frame's kernel identity alias (`addr_store_load`); `agent_chan_send_bytes(agent_c, ep_a, 0x5A5A, 0, buf_va, 1024)` -> expect Ok; `agent_chan_recv_bytes(agent_d, ep_b, buf_va_d, 1024)` -> expect Ok + bytes-len 1024; read agent_d's buffer back through ITS frame's identity alias and assert byte-for-byte equality with the pattern AND assert agent_c's and agent_d's buffer PAs DIFFER (proves it genuinely crossed address spaces, not an alias). Then exercise the fail-closed paths: a send with `len > MAX_PAYLOAD` -> Denied (nothing enqueued); a `copy_to_user` to an UNMAPPED dst VA -> Fault with the message NOT consumed (recv again with the good buffer still delivers it); a recv with `dst_cap` smaller than the queued byte_len -> Fault without discard. Print `tb_hal::serial_write_str("ipc: byte payload copy_to/from_user round-trips across two agent address spaces (distinct PAs)
")` then `tb_hal::serial_write_str("M14.1: payload OK
")`. Keep the cumulative `M14: ipc OK` line immediately after, unchanged. A new `m14_fail(...)` on any mismatch halts BEFORE M14.1/M14 print, so the cumulative chain (and the final CI marker) never appears -> fail-closed.
8. 8. Factor the leaf-flag policy into a pure `fn classify_leaf(flags, write) -> Result<(), CopyFault>` and a `fn span_pages(uva, len) -> impl Iterator` length helper in each uaccess.rs so the BOUNDS/permission logic is allocation-free pure functions; add a `#[cfg(kani)]`/unit-test harness asserting classify_leaf rejects every (not-present, not-user, write-on-RO) combination and span_pages never produces an out-of-range slice. The raw walk + physaccess is proven by the boot self-test (the arbiter); the safe classifier is the statically verifiable core.

## Decisions — no open questions (açık karar bırakma)
- Cross-space copy mechanism: a SOFTWARE page-table walk against an explicit root_pa, accessing the translated frame through the kernel SUPERVISOR identity/physmap alias -- NOT a CR3/TTBR0 switch and NOT a hardware access through the user VA. Why: it works when neither agent root is live (the disarmed-timer self-test runs in the boot root), needs zero TLB/CR3 churn on a single core, and is SMAP/PAN-immune by construction because the identity alias is a supervisor (U/S=0) mapping.
- Bounce buffer over zero-copy for M14: the byte payload travels as a kernel-heap `Box<[u8]>` parked in the Message (one copy in at send, one copy out at recv). Why: agent IPC messages are small control/coordination blobs (tool-call JSON, tokens, short prompts) where two memcpys are dwarfed by the scheduling/dispatch cost, and bounce buffering needs no page pinning, no TLB shootdown, and no remap; bulk/zero-copy sharing is already served by M15 shared memory blocks (`map_in_space`), so M14 and M15 are the two honest halves of the design.
- Payload bound: `ipc::MAX_PAYLOAD = 4096` bytes (one page); a send over the bound returns `SysStatus::Denied`. Why: a one-page cap keeps worst-case kernel-heap pressure bounded (MAX_PAYLOAD x ring depth) and matches seL4's one-page IPC buffer; anything larger is explicitly the M15 shared-block path.
- copy primitive signatures: `copy_from_user(root_pa, src_uva, dst: &mut [u8]) -> Result<(), CopyFault>` and `copy_to_user(root_pa, dst_uva, src: &[u8]) -> Result<(), CopyFault>`, length carried by the slice. Why: the explicit `root_pa` (not an implicit live CR3) is exactly what a framekernel needs to copy between two non-live spaces; Rust slices carry the bound so there is no separate length arg to get wrong.
- Fault type: `CopyFault { NotPresent, NotUser, NotWritable, NotInPhysmap }` (pure Copy enum in arch/mod.rs). Why: a typed, closed fault (never an errno/negative int), one variant per real walk failure, fail-closed by default; it maps to the single IPC-level `SysStatus::Fault`.
- IPC failure status: add `SysStatus::Fault = 9` AFTER `PeerClosed = 8`, keeping the closed status enum total. Why: a copy fault is semantically distinct from WouldBlock/Denied/PeerClosed and the receiver/sender must be able to tell them apart; appending preserves every existing discriminant.
- Send atomicity: `copy_from_user` runs BEFORE the carried-cap `detach`, and outbox-space + size are checked first; any failure before push_back leaves the sender's table and the ring untouched. Why: preserves the M11/M14 all-or-nothing invariant -- a copy fault can never strand a detached-and-lost capability.
- Recv atomicity: peek the head's byte length first (too-small dst_cap -> Fault WITHOUT pop, Zircon BUFFER_TOO_SMALL-no-discard semantics); otherwise pop, attempt `copy_to_user`, and on a copy fault `push_front` the whole Message back (restoring FIFO order + the parked cap + bytes). Why: a faulting or undersized receiver buffer never silently drops an in-flight message -- the message survives for a retry with a valid buffer.
- Unsafe confinement: the page-table walk + raw physical access live ONLY in the two NEW `crates/tb-hal/src/arch/{x86_64,aarch64}/uaccess.rs` modules; `ipc.rs` and `caps.rs` stay `#![forbid(unsafe_code)]` (the Message merely owns a safe `Box<[u8]>`), and the kernel crate stays `#![forbid(unsafe_code)]`, only calling the safe facades. Why: the framekernel rule that all unsafe + asm is confined to tb-hal.
- Driving path: the self-test drives the copy through the kernel facades `agent_chan_send_bytes`/`agent_chan_recv_bytes` (which resolve `agent_root_pa`) with the timer disarmed; the real ring3/EL0 user-syscall path is wired through `agent_syscall_current` (already knows the current agent and can supply its root) reading `args[2..3]` as the (uva,len) the roadmap reserved -- this user-syscall step is the additive layer, like M14's deferred blocking-recv. Why: an address-space-dependent op must ride the kernel facade exactly as M15's `M_BLOCK_MAP` does, and a disarmed-timer self-test cannot execute user instructions.
- Proof marker: print a NEW greppable line `M14.1: payload OK` immediately before the unchanged cumulative `M14: ipc OK`; fail-closed `m14_fail` halts before either prints on any byte mismatch. Why: an independently assertable sub-marker in the existing L2.x house style, while the cumulative re-run chain and the existing single-marker CI grep stay intact.
- Copy granularity: byte-wise `copy_nonoverlapping` over the per-page slice, never a u64 cast over the user buffer. Why: an arbitrary user VA is not naturally aligned; byte copies avoid the unaligned-access faults the Linux unaligned-memory-access doc warns about, on both targets.

## Risks & mitigations
- TOCTOU between the permission walk and the copy: mitigated by single-core + interrupts-masked execution (the same discipline the channel RefCell rings already rely on) and by fusing check+copy per page in one primitive -- the page tables provably cannot change between the walk and the access.
- Translated frame outside the identity/physmap window (e.g. a future allocator that hands out high frames): mitigated by the explicit `NotInPhysmap` fault that asserts the leaf PA is inside the boot identity window before any access -- fail-closed, never a wild supervisor deref.
- Huge-page or unexpected leaf in the user range silently mis-copied: mitigated by treating any PS/block descriptor in the user walk as NotPresent (TABOS maps only 4 KiB user leaves), so an unexpected layout faults instead of copying garbage.
- Receiver-side message loss on a bad/small buffer: mitigated by peek-before-pop for too-small buffers and push_front-restore on a copy fault, so an in-flight message is never silently dropped.
- Carried-cap stranding if a copy fault interleaves with a detach: mitigated by ordering copy_from_user strictly BEFORE the cap detach on send, so all pre-enqueue failure paths leave the sender's table intact (M11 all-or-nothing preserved).
- aarch64/x86 AP vs U/S bit divergence causing a one-arch-only bug: mitigated by a single shared CopyFault contract, symmetric self-test on both lanes, and citing the exact bit positions (x86 U/S bit2 every level; aarch64 AP[2:1]=bits[7:6], AF bit10) transcribed from the in-repo mmu.rs that the green M10/M12 mappers already use.
- Kernel-heap pressure from many large parked buffers: bounded by MAX_PAYLOAD (4096) x ring depth (the channel `bound`), and any oversize send is Denied -- the heap cost is statically capped.

## Research notes (with sources)
- Linux copy_to_user/copy_from_user return the number of bytes that could NOT be copied (0 == complete success); the user range is first validated by access_ok(), and on x86 the actual access is bracketed by STAC/CLAC. TABOS mirrors the 'validate-then-copy, report-the-shortfall' shape via the per-page walk + CopyFault (Linux include/linux/uaccess.h; the SMAP STAC/CLAC bracketing is the Linux uaccess implementation merged in 3.7).
- Intel SMAP: when CR4.SMAP (bit 21) is set, a supervisor (CPL<3) explicit access to a user-mode page page-faults unless EFLAGS.AC is set (toggled by the STAC/CLAC instructions); merged into Linux 3.7 (Dec 2012), enabled by default where supported. TABOS sidesteps this entirely by copying through the kernel's SUPERVISOR identity alias of the frame (U/S=0), never through the user VA -- so SMAP/PAN cannot fire (verified: Wikipedia 'Supervisor Mode Access Prevention' citing Intel SDM Vol.3A; corroborated by the existing M4 user.rs note that an identity-address write is SMAP-immune).
- Intel SDM Vol.3A 4.6 'Access Rights': a user-mode (and the equivalent supervisor walk validation) access is permitted only if U/S=1 in EVERY paging-structure entry controlling the translation -- so the x86 walk must check bit2 at PML4E/PDPTE/PDE/PTE, not just the leaf (transcribed in the in-repo crates/tb-hal/src/arch/x86_64/user.rs module header; Linux _PAGE_BIT_USER=2, _PAGE_BIT_RW=1, _PAGE_BIT_PRESENT=0).
- Arm VMSAv8-64 leaf encoding: AP[2:1] = descriptor bits[7:6] (AP[1]/bit6 = EL0 access enable, AP[2]/bit7 = read-only), AF = bit[10] mandatory on every leaf, page descriptor low bits = 0b11 (verified in the in-repo crates/tb-hal/src/arch/aarch64/mmu.rs header, which transcribes the Arm ARM and cross-checks Linux pgtable-hwdef.h PTE_AF/PTE_UXN). aarch64's PAN (PSTATE.PAN) is the SMAP analog and is likewise sidestepped by accessing via the EL1 identity alias.
- rust-vmm vm-memory exposes exactly the bounds+permission-then-access pattern this design fuses: `GuestMemory::check_range(addr, count, access) -> bool` validates a range is accessible under a Permissions mode, and `get_slices(addr, count, access)` returns VolatileSlices that may be FRAGMENTED across multiple underlying regions (the cumulative length equals count, and the first error ends iteration). TABOS's per-page walk loop mirrors this fragmented, permission-checked access (verified at docs.rs/vm-memory GuestMemory trait).
- Byte-granular copy is the portable choice: the Linux unaligned-memory-access doc states a single-byte (u8) access can never be unaligned and recommends memcpy over u8* to avoid unaligned faults on architectures that trap them -- so copy_to/from_user copies byte-wise over the per-page slice rather than casting the user buffer to u64 (verified at kernel.org core-api/unaligned-memory-access).
- seL4 IPC bounds the payload: short messages travel in message registers (seL4_MsgMaxLength words), longer data via a fixed per-thread one-page IPC buffer -- i.e. a bounded copy, not arbitrary zero-copy; this grounds the MAX_PAYLOAD=4096 one-page bounce-buffer decision and the 'bulk -> shared memory' split (seL4 Reference Manual, IPC chapter).
- In-repo precedent for the orchestration split: M15 already performs cross-address-space work (frame zeroing via the safe `addr_store_load` identity facade) and routes the address-space-dependent `M_BLOCK_MAP` through the kernel facade `agent_block_map` rather than `caps::dispatch`, while M10's `map_user_in_root`/`address_space_new` and lib.rs `agent_root_pa` already give the exact root-resolution + user-page-mapping primitives copy_to/from_user need (verified in crates/tb-hal/src/{blocks.rs,lib.rs} and arch/*/mmu.rs).
